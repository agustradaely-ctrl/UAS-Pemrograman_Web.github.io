Pertemuan 9

Form Handling & Validation

Pertemuan ini membahas cara menangani data form dan melakukan validasi input menggunakan PHP (server-side) dan JavaScript (client-side).

Form Method (GET vs POST)

Method GET mengirim data melalui URL dan cocok untuk data tidak sensitif. POST mengirim data melalui body request dan lebih aman untuk data pengguna.

Source Code:
<form method="post" action="proses.php">
    <input type="text" name="nama">
    <button type="submit">Kirim</button>
</form>

Validasi Client-side

Validasi client-side dilakukan di browser menggunakan JavaScript untuk mencegah pengiriman data kosong atau tidak valid.

Source Code (JavaScript):
<script>
function validasiForm() {
    const nama = document.getElementById("nama").value;
    if (nama === "") {
        alert("Nama wajib diisi");
        return false;
    }
    return true;
}
</script>

Validasi Server-side

Validasi server-side dilakukan menggunakan PHP untuk memastikan data tetap aman meskipun validasi client-side dilewati.

Source Code (PHP):
<?php
if (empty($_POST["nama"])) {
    echo "Nama tidak boleh kosong";
}
?>

Sanitasi Input

Sanitasi input bertujuan membersihkan data dari karakter berbahaya guna mencegah serangan seperti XSS dan SQL Injection.

Source Code:
<?php
$nama = htmlspecialchars($_POST["nama"]);
?>

File Upload

PHP menyediakan mekanisme upload file menggunakan atribut enctype="multipart/form-data".

Source Code:
<form method="post" enctype="multipart/form-data">
    <input type="file" name="gambar">
    <button type="submit">Upload</button>
</form>
